# Restaurant Club App - PRD
**Tagline: "Who's picking this time?"**

## Overview
A social tracking app for restaurant clubs to coordinate monthly dining adventures, track attendance, share reviews, and build a collective dining history.

## Core User Flow
1. **Club Setup**: Create club with zip code (shows other clubs within 25 miles)
2. **Monthly Cycle**: Member picks restaurant ‚Üí Poll/confirm ‚Üí Book via OpenTable ‚Üí Dine ‚Üí Review/photos ‚Üí Repeat
3. **Social Layer**: Comments, mentions, photo sharing throughout the experience

## User Types
- **Eaters**: Club members who participate in monthly dinners
- **Club Admin**: Creates club, manages members (optional role)

## Technical Requirements

### Data Models
```
Club {
  id, name, zipCode, createdAt
  members: [User]
  restaurants: [RestaurantEntry]
  wishlist: [Restaurant]
}

User {
  id, name, email, avatar
  clubs: [Club]
}

RestaurantEntry {
  id, restaurant, picker, month, year
  totalBill, tags: [String]
  attendees: [Attendance]
  photos: [Photo]
  comments: [Comment]
  clubRating: Number
  reservation: {
    openTableId, date, time, partySize
    confirmationNumber, status
  }
}

Attendance {
  userId, attended: Boolean
  individualRating: Number
  review: String
}
```

### Core Integrations
- **Google Maps API**: Restaurant search, map display
- **Google Places API**: Restaurant details, reviews
- **OpenTable API**: Restaurant availability, reservations, menus
  - Check real-time availability for party size
  - Browse available time slots
  - Direct booking from app
  - Reservation management (modify, cancel)
  - Confirmation and reminder emails
  - Restaurant details, photos, and menus
  - Hours and contact information
- **Push Notifications**: Reminders and social notifications
- **Image Storage**: Photo uploads with compression

## UI/UX Guidelines
- **Vibe**: Fun, friendly, social
- **Colors**: Food-inspired palette (warm oranges, deep reds, fresh greens)
- **Typography**: Casual but readable
- **Photography**: Emphasize food photos, group shots
- **Interactions**: Swipe between restaurants, tap to expand details
- **Booking Flow**: Seamless 2-3 tap reservation process

## Success Metrics
Since this is for personal use:
- Daily active usage by club members
- Photos uploaded per restaurant
- Comments/engagement per entry
- Picker compliance rate (people actually choosing on time)
- Reservation completion rate via OpenTable
